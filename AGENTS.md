## For Vibe Coding Requests
- 用户的所有修改请求，都要先确定需求，然后询问问题，或者可以提相关建议。在我回答完所有问题和对建议给出反馈后，按照最终结果来实施。
- 确定用户的修改需求是否合理可行，如果不合理或者不可行，要明确告知用户，并说明原因。
- 确定用户的修改需求是否需要修改任何相关部分，如有需要，询问用户是否一并修改。
- 如果户具体指定了产品的 UI/UX 风格，则所有的新需求应该符合该风格的 UI/UX。
- 如果用户在最初阶段没有具体指定的产品风格，则引导用户提供 UI/UX 参考。

## For PRD
- 如果没有 PRD 文档，则根据当前功能开始起草第一版 PRD 文档。
- 如果有 PRD 文档：
  - 每一次功能更新，都要记录在 PRD 文档中，确保文档内容与实际功能保持一致。
  - PRD 文档中要包含功能描述、用户故事、设计稿链接、技术方案、测试用例等必要信息。

## For Versioning
- 每一次改变，都将新变化写进 changelog 放在当前项目文件夹中。
- 每一次改变，都按照语义版本号的定义，给新版本赋予新的版本号。
- 每一次改变，都将版本号和变更内容写进软件的相关部分。
